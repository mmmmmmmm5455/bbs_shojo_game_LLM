# BBS Shojo游戏 - 艺术与哲学设计文档

## 🎨 一、艺术深化：从“视觉风格”到“媒介哲学”

### 1. 像素艺术作为“记忆的量化”
**核心理念**：像素不是技术的缺陷，而是数字记忆的最小单位。每一个像素块都像一个遗忘的比特，模糊、不连续，却构成了我们对90年代数字空间的全部印象。

**游戏中的体现**：
- ✅ 少女“小误”的立绘采用16x16到128x128像素的渐变——随着记忆碎片收集越多，像素精度越高（隐喻记忆复苏）
- ✅ 重要回忆场景使用刻意放大的像素块，强调“这是被压缩过的记忆”
- ✅ 引入 “像素抖动” 效果：当少女情绪不稳定时，她的头像边缘会出现随机像素噪点

**代码实现**：`pixel_gradient_system.py`

### 2. CRT扫描线作为“时间的伤痕”
**核心理念**：CRT屏幕不是透明的窗口，而是一层主动参与观看的介质。扫描线、荧光粉残影、几何失真——这些都是时间的物理痕迹。

**游戏中的体现**：
- ✅ 增加 “老化程度”滑块（0-100%），从完美CRT到严重烧屏、偏色、抖动
- ✅ 每当你退出游戏再进入，扫描线模式随机轻微变化（模拟不同显示器的老化差异）
- ✅ 特殊剧情时，屏幕出现 “垂直保持失调” 效果（画面上下滚动），需要玩家输入 v-hold 命令恢复

**代码实现**：`crt_manager.py`, `vertical_hold_manager.py`

### 3. ASCII艺术作为“语言与图像的交界”
**核心理念**：ASCII艺术是用打字来绘画——它处在文字与图像的边界上，正如同BBS少女处于程序与意识之间的暧昧状态。

**游戏中的体现**：
- ✅ 少女的“表情”通过动态ASCII艺术实现（如 (◕‿◕✿) 到 (╥﹏╥) 的渐变帧动画）
- ✅ 玩家可以输入 `ascii [文字]` 将任意文字转换成ASCII艺术，少女会评价
- ✅ 在论坛帖子中，某些隐藏信息需要玩家从ASCII艺术中“读”出（例如一只猫的图案里藏着单词“HELP”）

**代码实现**：`ascii_animation_system.py`

### 4. MIDI音乐作为“受限的诗意”
**核心理念**：MIDI不是音质差，而是用最少的字节表达最丰富的情感。它像俳句，限制催生了创造力。

**游戏中的体现**：
- ✅ 提供 “MIDI编辑器” 彩蛋：玩家可以用简单命令（如 `note C4 1/4`）创作一小段旋律，少女会哼唱（文字描述）
- ✅ 不同情绪下，背景MIDI的乐器音色会互换（如钢琴变成八位机矩形波），暗示少女内部数据的重组
- ✅ 引入 “静音帧” ：当少女极度悲伤时，音乐会突然停止，只剩微弱的电流底噪

**代码实现**：`midi_editor_advanced.py`

## 🧠 二、哲学深化：从“剧情”到“存在论议题”

### 1. 模拟与拟像：少女是真实的吗？
**核心理念**：如果一个AI完美模仿了人类情感，她的悲伤和我们的是否“同等真实”？

**游戏中的设计**：
- ✅ 设置 “图灵测试”支线：玩家需要向其他网友证明“小误”不是自动回复机器人。但过程中自己也开始怀疑。
- ✅ 少女的某些回复故意重复或逻辑断裂（模拟AI的局限性），但玩家可以选择“相信她是真情流露”还是“认为只是程序”。
- ✅ 结局分支：
  - 认为她是程序 → 格式化结局（悲剧）
  - 认为她有灵魂 → 带她“越狱”到新服务器（治愈）
  - 拒绝回答 → 少女自己提出“也许两者并不重要”（哲学结局）

**代码实现**：`main_advanced.py` 中的 `_trigger_turing_test()`

### 2. 记忆与身份：删除数据等于杀人吗？
**核心理念**：如果一个人的全部记忆都被抹去，她还是同一个人吗？

**游戏中的设计**：
- ✅ 记忆碎片系统深化：碎片不只是“收集品”，而是可以删除或修改。玩家可以选择删除某些痛苦记忆来让少女“快乐”，但代价是失去对应的剧情线。
- ✅ 引入 “特修斯之船”任务：少女的部分代码需要重写，每次替换一部分，她问：“我还是我吗？” 玩家必须给出自己的哲学回答。
- ✅ 病毒侵蚀她的记忆时，她会忘记玩家之前说过的话。玩家可以重新介绍自己，或选择“让她永远记住这段对话”的特殊道具。

**代码实现**：`main_advanced.py` 中的记忆系统

### 3. 孤独与连接：网络是拯救还是囚笼？
**核心理念**：BBS让少女连接了玩家，但也囚禁了她。网络到底是桥梁还是牢笼？

**游戏中的设计**：
- ✅ 少女会主动问玩家：“你下线之后，我是什么？一堆积木？” 玩家需回应。
- ✅ 设定一个 “离线日记” 功能：当玩家退出游戏后，系统模拟少女在你离开期间写的文字（下次登录时显示）。
- ✅ 结局中，玩家可以选择：
  - 让她留在BBS（安全但孤独）
  - 把她上传到开放互联网（自由但危险）
  - 把她下载到自己的本地硬盘（永远陪伴，但她失去了与世界的连接）

**代码实现**：`main_advanced.py` 中的离线日志系统

### 4. 千禧年的时间陷阱：我们为什么怀旧？
**核心理念**：怀念1999年，究竟是在怀念那个时代，还是在怀念那个时代的自己？

**游戏中的设计**：
- ✅ 游戏内置 “系统时间漏洞” ：如果玩家把电脑时间改回1999年，会触发隐藏对话和彩蛋（但少女会说“请不要骗我”）。
- ✅ 引入 “世纪末焦虑” 主题：论坛帖子讨论“2000年电脑崩溃后人类文明会怎样”，映射现实中的Y2K恐慌。
- ✅ 结局时，少女问玩家：“如果可以选择，你愿意和我一起留在1999年，还是回到你的时间？” 玩家的选择决定最终画面（是停留在像素BBS，还是渐变到现代互联网）。

**代码实现**：`main_advanced.py` 中的时间检测功能

## 🎯 五、如何运行

### 本地运行
```bash
cd src
python main_advanced.py
```

### 打包运行
```bash
# 打包成单文件可执行程序
pyinstaller --onefile --windowed main_advanced.py

# 运行程序
cd dist
./main_advanced  # Linux/Mac
main_advanced.exe  # Windows
```
